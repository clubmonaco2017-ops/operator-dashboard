### Stage 2 — Iconography sweep (custom Icon functions + emoji → lucide-react)

**Цель:** Per spec D-5 + §4.1, eliminate ALL custom inline `function XxxIcon()` definitions and most emoji from production code, replacing with `lucide-react` named imports. ~37 component files affected (broader than spec's "~20" estimate; verified via `grep -rln "function.*Icon\s*("`). Direct imports per D-7 (no central `src/lib/icons.js`).

**Files:** see "File structure → Modify — JSX" above. Sweep grouped by folder for cohesive commits.

**Strategy — for EACH file (general pattern):**

1. Open file. Identify all local `function XxxIcon()` / `function Spinner()` definitions (usually at bottom).
2. Add `import { LucideName1, LucideName2 } from 'lucide-react'` at top (next to other imports).
3. Replace each `<XxxIcon />` JSX with `<LucideName size={N} className="..." />` keeping any wrapping props.
4. **Delete** the local `function XxxIcon()` definitions entirely.
5. If a deleted function had `props.className` forwarding or other side-effects unique to it, transfer them to the Lucide replacement (Lucide accepts `className`, `style`, `strokeWidth`, `size`, `color`).
6. Re-render the affected route in dev server; visually check icon renders at correct size and stroke.

## Rollback / debugging notes

- **If Stage 1 visual smoke shows token regression** (e.g., a color disappears on a route): the `@theme inline` additions are pure additions — they cannot remove existing utilities. Most likely cause is the universal `:focus-visible` rule overriding a custom focus style. To diagnose, comment out the new `:focus-visible` block and re-test. If that's the cause, the affected component needs a higher-specificity override (or accept the new uniform style).

- **If Stage 2 sweep breaks an icon's visual size** — Lucide icons default to `size={24}` (24×24 px viewBox via `width`/`height` defaults) — much larger than typical 12-16 px contexts. ALWAYS pass `size={N}` per mapping table. If still misaligned, check parent flex/inline alignment (Lucide is inline-block by default).

- **If Stage 3 install adds a TypeScript file** to a JSX-only project — that's a registry/config drift. Either rename to `.jsx` and strip types (small primitives are easy) or roll the file back and ask user how to proceed.

- **If `git push` fails on `feat/subplan-6-design-system`** because branch already exists on origin — that means a previous attempt was made. Diff local vs remote: `git fetch && git log feat/subplan-6-design-system..origin/feat/subplan-6-design-system`. If remote has commits not local: stop, ask user. If local is ahead and remote unchanged: `git push -u origin feat/subplan-6-design-system --force-with-lease` (after user confirms — never force-push without confirmation).
